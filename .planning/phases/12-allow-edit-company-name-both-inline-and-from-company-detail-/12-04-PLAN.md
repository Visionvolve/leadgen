<objective>
Ship the merge endpoint and its underlying transactional helper. When
the user picks "Merge into this one" in the modal, the frontend calls
POST /api/companies/<id>/merge?into=<surviving_id>, the server verifies
both rows are in the caller's tenant, locks them FOR UPDATE, fills NULL
slots on the surviving row from the duplicate, re-points every FK to the
deleted company, writes one audit row with a JSON snapshot of the
deleted record, and hard-deletes the duplicate — all in one transaction.

Purpose: Without this, the modal's "Merge into this one" action has no
backend. Plans 12-05/06/07 wire the UI to this endpoint.
Output: One new route, one new helper in dedup.py, one new test module.
</objective>

POST /api/companies/<id>/merge?into=<surviving_id>:
  - Auth: editor + same tenant for BOTH ids
  - 200: returns surviving company payload (dict)
  - 400: invalid uuid OR ids identical OR missing `into` query param
  - 404: either company not in caller's tenant
  - 500 (rolled back): FK re-point failure

      import json
      from sqlalchemy import text as sa_text
      from .contact_helpers import write_field_change

      # Tables whose `company_id` column references companies(id).
      # Discovered via `awk '/__tablename__/{t=$0} /ForeignKey..."companies.id"/{print t}'`.
      # Each entry: (table_name, is_pk_table). PK tables can have at most
      # one row per company_id; on conflict we keep surviving's row and
      # delete the deleted's row instead of UPDATE.
      _COMPANY_FK_TABLES: list[tuple[str, bool]] = [
          ("contacts",                       False),
          ("company_tags",                   True),
          ("company_tag_assignments",        True),
          ("company_enrichment_l1",          True),
          ("company_enrichment_l2",          True),
          ("company_enrichment_market",      True),
          ("company_enrichment_opportunity", True),
          ("company_enrichment_profile",     True),
          ("company_enrichment_signals",     True),
          ("company_legal_profile",          True),
          ("company_news",                   False),
          ("company_registry_data",          True),
          ("company_insolvency_data",        True),
          ("strategy_documents",             False),
      ]

      # Surviving fields that win when non-NULL on surviving; otherwise
      # fall back to the deleted row's value. Mirrors the
      # `update_empty_fields` pattern. Columns intentionally excluded:
      # id, tenant_id, created_at, updated_at, name, normalized_name
      # (name is the operator's last-saved value on surviving; we never
      # auto-promote the deleted name).
      _FILLABLE_FIELDS: list[str] = [
          "domain", "website_url", "linkedin_url", "logo_url", "notes",
          "summary", "triage_notes", "industry", "industry_category",
          "geo_region", "hq_city", "hq_country", "tier", "status",
          "owner_id", "tag_id", "business_model", "company_size",
          "ownership_type", "revenue_range", "buying_stage",
          "engagement_status", "crm_status", "ai_adoption",
          "news_confidence", "business_type", "cohort", "ico",
          "official_name", "tax_id", "legal_form", "registration_status",
          "date_established", "credibility_score",
          "verified_revenue_eur_m", "verified_employees",
          "organization_type", "segment",
      ]

      class MergeError(ValueError): pass

      def merge_companies(
          tenant_id: str,
          deleted_id: str,
          surviving_id: str,
          changed_by: str | None,
      ) -> dict:
          if deleted_id == surviving_id:
              raise MergeError("Cannot merge company into itself")

          # Lock both rows in the same transaction. ORDER BY id keeps
          # deadlocks deterministic.
          rows = db.session.execute(sa_text("""
              SELECT id, tenant_id, name, normalized_name, """ + ", ".join(_FILLABLE_FIELDS) + """
              FROM companies
              WHERE id IN (:a, :b) AND tenant_id = :t
              ORDER BY id
              FOR UPDATE
          """), {"a": deleted_id, "b": surviving_id, "t": str(tenant_id)}).mappings().all()

          if len(rows) != 2:
              raise MergeError("Both companies must exist in caller's tenant")

          by_id = {r["id"]: dict(r) for r in rows}
          deleted = by_id[deleted_id]
          surviving = by_id[surviving_id]

          # Snapshot for audit BEFORE we mutate anything
          snapshot = {k: (str(v) if v is not None else None) for k, v in deleted.items()}

          # 1. Fill NULL fields on surviving from deleted
          fills = {}
          for f in _FILLABLE_FIELDS:
              if surviving.get(f) in (None, "") and deleted.get(f) not in (None, ""):
                  fills[f] = deleted[f]
          if fills:
              set_parts = ", ".join(f"{k} = :{k}" for k in fills)
              params = dict(fills)
              params["id"] = surviving_id
              db.session.execute(sa_text(
                  f"UPDATE companies SET {set_parts} WHERE id = :id"
              ), params)

          # 2. Re-point FK tables
          for table, is_pk in _COMPANY_FK_TABLES:
              if is_pk:
                  # Drop deleted row when surviving already has a PK row in this table
                  db.session.execute(sa_text(f"""
                      DELETE FROM {table}
                      WHERE company_id = :d
                        AND EXISTS (SELECT 1 FROM {table} WHERE company_id = :s)
                  """), {"d": deleted_id, "s": surviving_id})
                  # Move remaining (no conflict)
                  db.session.execute(sa_text(f"""
                      UPDATE {table} SET company_id = :s WHERE company_id = :d
                  """), {"d": deleted_id, "s": surviving_id})
              else:
                  db.session.execute(sa_text(f"""
                      UPDATE {table} SET company_id = :s WHERE company_id = :d
                  """), {"d": deleted_id, "s": surviving_id})

          # 3. Re-point semantic refs in audit table (not a FK in schema)
          db.session.execute(sa_text("""
              UPDATE contact_field_changes SET entity_id = :s
              WHERE entity_type = 'company' AND entity_id = :d
          """), {"d": deleted_id, "s": surviving_id})

          # 4. Write the merge audit row
          write_field_change(
              tenant_id=str(tenant_id),
              entity_type="company",
              entity_id=str(surviving_id),
              field_name="merged_from",
              old_value=str(deleted_id),
              new_value=str(surviving_id),
              changed_by=changed_by,
              source="merge",
              metadata={"deleted_snapshot": snapshot},
          )

          # 5. Hard-delete the duplicate
          db.session.execute(sa_text(
              "DELETE FROM companies WHERE id = :d AND tenant_id = :t"
          ), {"d": deleted_id, "t": str(tenant_id)})

          # 6. Return the surviving company payload
          new_surviving = db.session.execute(sa_text("""
              SELECT id, tenant_id, name, normalized_name, domain, status, owner_id, notes
              FROM companies WHERE id = :s
          """), {"s": surviving_id}).mappings().fetchone()
          return dict(new_surviving) if new_surviving else {}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → merge endpoint | Untrusted POST body + query string |
| tenant scoping | Both ids resolved against resolve_tenant() — not payload |
| FK integrity | DELETE on companies after all FKs re-pointed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-18 | I (Cross-tenant merge / data destruction) | merge_company route | mitigate | Both ids loaded via WHERE tenant_id=:t in merge_companies; len(rows) != 2 raises MergeError → 404. Test case 4 enforces. |
| T-12-19 | T (Concurrent edit race) | two operators merging same companies | mitigate | SELECT ... FOR UPDATE on both rows; second operator's transaction blocks until first commits, then re-reads and finds rows missing → MergeError. |
| T-12-20 | D (Data destruction without recovery) | hard-delete | mitigate | metadata.deleted_snapshot captures every column of the deleted row before deletion. Recovery path is human-driven from audit log; documented in spec. |
| T-12-21 | T (Audit log injection via merged_from) | metadata JSON | mitigate | Snapshot built server-side from typed query result; no client-controlled JSON merged. |
| T-12-22 | E (Privilege escalation) | endpoint authz | mitigate | @require_role("editor") preserved. Test case 7 enforces. |
| T-12-23 | T (FK orphans from partial failure) | transaction integrity | mitigate | All operations inside Flask's request-bound session; explicit db.session.rollback() in except path; test case 9 enforces. |
| T-12-24 | T (SQL injection via dynamic table names) | _COMPANY_FK_TABLES loop | mitigate | Table names are a static Python list, not request input; f-string interpolation is safe because the values come from a hard-coded literal list. |
| T-12-25 | I (Snapshot leak in metadata) | metadata.deleted_snapshot | accept | Snapshot is only readable by operators with audit-log access; same trust level as the row itself. |
</threat_model>
