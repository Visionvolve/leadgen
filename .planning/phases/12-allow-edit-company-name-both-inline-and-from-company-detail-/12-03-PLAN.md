<objective>
Add the server-side duplicate gate to the existing PATCH /api/companies/<id>
endpoint. When the payload changes `name`, compute the normalized form,
reject empty, look up same-tenant collisions, and either (a) 409 the
client with match summaries, or (b) — when `?confirm_duplicate=keep_both`
is set — write the rename plus an annotated audit row.

Purpose: This is the back half of the inline-edit flow. The frontend
(plans 12-05/06/07) reads the 409 body and opens the resolution modal.
Output: Modified PATCH handler, extended write_field_change to accept
metadata, and integration tests covering five scenarios (clean rename,
empty after normalize, collision, keep-both, tenant isolation).
</objective>
